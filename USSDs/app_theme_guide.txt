## ANIMATION & TRANSITIONS

### Transition Types
- Button press: 0.1s ease-out
- Screen transitions: 0.3s ease-in-out
- List item animations: 0.2s ease
- Loader animations: Infinite 1.5s linear

### Animation Guidelines
- Subtle, purpose-driven animations
- Use motion to communicate state changes
- Keep animations under 400ms for responsiveness
- Use spring physics for natural movement

### Key Animations
- Code execution loading: Pulse animation with primary colors
- Success/error feedback: Gentle bounce effect
- Category switching: Slide transition
- Bottom navigation: Fade combined with small vertical shift

## ACCESSIBILITY CONSIDERATIONS

### Color Contrast
- All text meets WCAG AA standards (4.5:1 for normal text, 3:1 for large text)
- Interactive elements have distinct focus states
- Critical information never conveyed by color alone

### Text Sizing
- All text can scale up to 200% without breaking layouts
- Minimum touch target size of 44x44 pixels

### Dark Mode Implementation
- True dark mode (not just inverted colors)
- Reduced brightness for eye comfort
- Careful use of dark blues instead of pure black for less eye strain

### Alternative Interactions
- Voice control support for code execution
- Haptic feedback for button presses
- Screen reader friendly labels for all interactive elements
